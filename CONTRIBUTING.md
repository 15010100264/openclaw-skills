# 贡献指南

欢迎为OpenClaw技能中心贡献技能！本指南将帮助你快速上手。

## 贡献流程

### 1. 准备工作
1. **Fork仓库**：Fork [openclaw-skills](https://github.com/openclaw-skills) 组织中的仓库
2. **克隆仓库**：`git clone https://github.com/你的用户名/仓库名`
3. **安装依赖**：`npm install`（如果需要）

### 2. 开发技能
1. **使用模板**：基于`skill-template`创建新技能
2. **遵循规范**：确保符合技能开发规范
3. **编写测试**：为技能编写测试用例
4. **更新文档**：完善SKILL.md和README.md

### 3. 提交贡献
1. **创建分支**：`git checkout -b feature/技能名称`
2. **提交更改**：`git commit -m "描述你的更改"`
3. **推送分支**：`git push origin feature/技能名称`
4. **创建PR**：在GitHub上创建Pull Request

### 4. 审核流程
1. **代码审查**：维护者会审查代码
2. **测试通过**：确保所有测试通过
3. **文档完整**：确保文档齐全
4. **合并发布**：审核通过后合并到主分支

## 技能开发规范

### 文件结构要求
```
技能名称/
├── SKILL.md          # 必须：技能描述文件
├── README.md         # 必须：详细文档
├── package.json      # 必须：项目配置
├── index.js          # 必须：主入口文件
├── test/             # 推荐：测试文件
│   └── index.test.js
└── docs/             # 可选：额外文档
    └── api.md
```

### SKILL.md规范
必须包含以下部分：
1. **描述**：技能功能简介
2. **使用场景**：适用场景列表
3. **安装**：安装命令示例
4. **使用方法**：工具列表和示例
5. **更新日志**：版本历史

### 代码规范
1. **使用ES6+语法**
2. **添加必要的注释**
3. **完善的错误处理**
4. **统一的返回格式**
5. **性能优化考虑**

## 技能分类

### 免费技能
- 许可证：MIT
- 要求：功能完整，文档齐全
- 审核：代码审查通过即可

### 付费技能
- 许可证：商业许可证
- 要求：高质量，有独特价值
- 审核：额外审核，确保商业可行性

### 企业技能
- 许可证：定制许可证
- 要求：企业级功能，技术支持
- 审核：企业需求评估

## 测试要求

### 单元测试
必须包含以下测试：
1. **初始化测试**：技能正确初始化
2. **工具测试**：每个工具的基本功能
3. **错误测试**：错误处理测试
4. **性能测试**：关键操作性能测试

### 测试覆盖率
- 最低要求：80%代码覆盖率
- 推荐目标：90%代码覆盖率

### 测试命令
```bash
# 运行测试
npm test

# 查看覆盖率
npm run coverage
```

## 版本管理

### 版本号规范
使用语义化版本号：
- **主版本号**：不兼容的API修改
- **次版本号**：向下兼容的功能新增
- **修订号**：向下兼容的问题修正

### 发布流程
1. **更新版本号**：`npm version patch/minor/major`
2. **更新日志**：更新CHANGELOG.md
3. **创建标签**：`git tag v1.0.0`
4. **发布Release**：在GitHub创建Release
